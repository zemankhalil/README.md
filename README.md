## Project Overview  
This project is a full stack web application that includes both a customer-facing interface and an administrative interface. The application allows users to interact with data dynamically, while administrators can securely log in to manage content. The final iteration includes secure authentication to protect administrative functionality.

---

## Architecture  

In this project, I worked with multiple frontend development approaches, including Express-based HTML rendering and a single-page application (SPA) using JavaScript.

The Express HTML approach uses server-side rendering, where pages are generated on the server and sent to the client. This method is simple but requires full page reloads. In contrast, the SPA approach loads a single page and dynamically updates content using JavaScript, improving performance and user experience.

The backend uses a NoSQL MongoDB database because it is flexible and stores data in a JSON-like format. This makes it easy to work with JavaScript and allows for scalability and faster development.

---

## Functionality  

JSON (JavaScript Object Notation) is a data format used to store and transfer information, while JavaScript is a programming language used to manipulate that data.

JSON connects the frontend and backend by serving as the format used when sending and receiving data through API calls. The frontend sends requests and the backend responds with JSON data that the frontend displays.

During development, I refactored code to improve structure and efficiency by organizing routes and reducing duplicate code. Creating reusable UI components helped improve consistency, reduced workload, and made updates easier across the application.

---

## Testing  

In a full stack application, HTTP methods like GET, POST, PUT, and DELETE are used to interact with API endpoints. Each endpoint allows the application to perform specific actions such as retrieving or updating data.

Testing ensures that endpoints return correct data and handle errors properly. With added authentication, testing becomes more complex because protected routes must verify user credentials before allowing access.

Security ensures that only authorized users can access sensitive parts of the application, such as the admin dashboard. This protects the system and user data.
